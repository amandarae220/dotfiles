# Monthly Audit — 2026-06-30 (habitTracker, d3examples)

## Executive Summary

Both repositories are small, dependency-free static projects. Neither has a build system, package manager, or shell scripts, so Sections 2 and 3 are N/A. Accessibility is the dominant concern across both repos: `habitTracker` has a broken stylesheet reference that prevents all styles from loading, an undefined function that crashes on every button click, and a focus-indicator suppression that locks out keyboard users. `d3examples` has pervasive missing `alt` attributes and an insecure HTTP script load. 11 findings total; 2 critical, 4 high, 3 medium, 2 low.

---

## 1. Skill Compliance

### 🚨 Critical

`A11Y [severity: critical]: habitTracker/style.css:30 — outline: none on input, button removes all keyboard focus indicators, violating WCAG 2.4.7 Focus Visible — Remove the rule or replace with a custom :focus-visible style (e.g. outline: 2px solid #005fcc)`

`A11Y [severity: critical]: d3examples/index.html:16, barChart.html:16, bubbleChart.html:16, force.html:16, globe.html:16, pieChart.html:16, svg.html:16 — <img src="resources/d3logo.png"> missing alt attribute on every page, violating WCAG 1.1.1 Non-text Content — Add alt="D3 Examples logo" (or alt="" if decorative)`

### 🔴 High

`QUALITY [severity: high]: habitTracker/list.js:128 — addEventListener("click", ajaxRequest) references a function that is never defined; every "Add Task" click throws a ReferenceError — Remove the line or implement the intended AJAX function`

`A11Y [severity: medium]: habitTracker/index.html:3 — <html> tag missing lang attribute, violating WCAG 3.1.1 Language of Page — Change to <html lang="en">`

`A11Y [severity: medium]: d3examples/ireland.html:2 — <html> tag missing lang attribute, violating WCAG 3.1.1 Language of Page — Change to <html lang="en">`

`A11Y [severity: high]: d3examples/barChart.html, bubbleChart.html, pieChart.html, force.html, svg.html — SVG charts have no accessible title, aria-label, or role, violating WCAG 1.1.1 — Add <title>Chart description</title> as first child of each <svg>, or set aria-label and role="img" on the <svg> element`

### 🟡 Medium

`A11Y [severity: medium]: habitTracker/list.js:19-22 — Dynamically created <input type="checkbox"> elements are never associated with their sibling <label> via for/id pairing, violating WCAG 1.3.1 Info and Relationships — Assign a unique id to each checkbox and set label.htmlFor to match`

`A11Y [severity: medium]: d3examples/*.html:21 — aria-label="..." on .btn-group is a literal placeholder string, not a meaningful description (WCAG 4.1.2) — Replace with aria-label="Page navigation" or similar`

`SECURITY [severity: medium]: d3examples/ireland.html:26 — D3 v2 loaded over http:// (http://d3js.org/d3.v2.js); modern browsers block mixed content and the plain-HTTP load is susceptible to MITM script injection — Change to https:// or vendor the file locally`

`A11Y [severity: medium]: d3examples/globe.html:80 — <canvas> element has no accessible fallback content and no aria-label; screen readers cannot convey the interactive globe (WCAG 1.1.1) — Add aria-label="Interactive rotating globe" and a <p> fallback inside the canvas`

### 🟢 Low

`QUALITY [severity: low]: habitTracker/index.html:7 — Stylesheet linked as href="styles.css" but the file on disk is style.css; all styles fail to load — Fix the href to style.css`

`QUALITY [severity: low]: d3examples/globe.html:6 — Commented-out <script> tag left in source — Remove the dead comment`

`A11Y [severity: low]: habitTracker/list.js:38-41 — Edit and Delete buttons provide no context about which task they act on; screen reader users hear "Edit, button" with no task name (WCAG 2.4.6) — Set aria-label dynamically: editButton.setAttribute("aria-label", "Edit " + taskString)`

`A11Y [severity: low]: habitTracker/index.html, d3examples/*.html — No <main> landmark element and no skip-navigation link; keyboard users must tab through header on every page load (WCAG 2.4.1) — Wrap primary content in <main> and add a visually-hidden "Skip to main content" anchor`

`ANTI-PATTERN: d3examples/index.html:76 — data = [...] assigned without var/let/const, creating an implicit global — Declare with const data = [...]`

`ANTI-PATTERN: d3examples/svg.html:120-122 — data1, data2, selectedData assigned without var/let/const, all leaking to global scope — Declare with const/let`

`ANTI-PATTERN: habitTracker/list.js:5 — document.getElementsByTagName("button")[0] selects the first button by tag index; fragile if markup order changes — Query by id or data attribute instead`

No `package.json` in either repo — versions are hardcoded CDN references. All third-party libraries are significantly outdated:

- **Bootstrap 3.3.6** (2015): EOL. Bootstrap 5 has breaking changes (dropped jQuery, renamed classes).
- **D3 v2** (`ireland.html`): 5 major versions behind. API has breaking changes in each major.
- **D3 v3** (all other chart pages): 4 major versions behind.
- **D3 v4.2.2** (`globe.html`): 3 major versions behind.

Recommended action: Consolidate to D3 v7 across all pages (requires API migration) and upgrade Bootstrap to v5.
