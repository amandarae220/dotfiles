# Monthly Audit — 2026-06-16 (amanda-repository, Calculator2.0)

## Executive Summary

Two repos audited this cycle. **amanda-repository** has the more urgent findings: a hardcoded admin password and live Supabase credentials are committed to source and shipped in the public client bundle, the unit test suite fails to compile, two divergent root components mean SSR and client-rendered output structurally differ, and the entire Angular toolchain is two majors behind with `npm audit` reporting 1 critical / 25 high vulnerabilities. **Calculator2.0** is in good shape overall (no secrets, no unsafe DOM APIs, solid a11y) but its GitHub Pages deploy workflow references a deleted branch (`feature/angular-makeover`) and will fail on next run, and a missing `preview.png` breaks social link previews.

A prompt-injection attempt was also investigated during this audit (see Security note below) — it was caught, not acted on, and turned out to be a false positive once the underlying file was checked directly.

## 1. Skill Compliance

### 🚨 Critical

- SECURITY [severity: critical]: amanda-repository/src/environments/environment.ts:5 — Plaintext admin password (`adminPassword: 'admininsights'`) committed to source control and shipped to the client bundle; checked client-side in `admin.component.ts:113` — anyone can read it from the compiled JS — fix: remove client-side password gating entirely; authenticate via a server-side endpoint or Supabase auth with hashed credentials.
- SECURITY [severity: critical]: amanda-repository/src/environments/environment.ts:3-4 — Live Supabase project URL and anon key committed in plaintext — fix: confirm Row Level Security policies on `portfolio_events` are airtight (anyone can currently call insert/select with this key); rotate if not.
- A11Y [severity: critical]: amanda-repository/src/app/pages/admin/admin.component.html:8-15 — Password input has no `aria-invalid`/`aria-describedby` association to the error message when login fails — screen readers won't announce the error — fix: add `[attr.aria-invalid]="authError"` and `aria-describedby` linking to the error paragraph.

### 🔴 High

- SECURITY [severity: high]: amanda-repository/portfolio-src.zip — 635KB zip archive committed at repo root, outside `.gitignore`, unknown contents — fix: inspect and remove from version control if not needed.
- QUALITY [severity: high]: amanda-repository/src/app/app.component.spec.ts:17-28 — Unmodified Angular CLI boilerplate spec references nonexistent `app.title`; `ng test` fails to compile, blocking the entire test suite — fix: rewrite spec to match real `AppComponent` or delete it.
- QUALITY [severity: high]: amanda-repository/src/app/app.component.ts vs src/app/root/root.component.ts — Two different root components share the selector `app-root`: `AppComponent` (used by `main.server.ts`, hardcodes all sections, bypasses routing) vs `RootComponent` (used by `main.ts`, uses `<router-outlet>`). SSR and client-rendered output are structurally different apps — fix: bootstrap the same root component from both entry points; delete the redundant one.
- 404 RISK (build-breaking): Calculator2.0/.github/workflows/deploy.yml:25 — "Checkout v2" step pins `ref: feature/angular-makeover`, which no longer exists (only `main` remains, already merged) — next workflow run will fail at checkout, breaking the v1/v2 split deploy — fix: point checkout at `main` for both v1/v2 or collapse to a single checkout.

### 🟡 Medium

- A11Y [severity: medium]: amanda-repository/src/app/sections/about/about.component.html:8 — Duplicate `<h1>` (About renders its own alongside Hero's `<h1>Hello!</h1>`) — breaks single-h1 document outline — fix: demote to `<h2>`.
- A11Y [severity: medium]: amanda-repository/src/app/sections/about/about.component.html:18 — `<button>VIEW INTERACTIVE RESUME</button>` has no handler, type, or routerLink — dead interactive element — fix: wire it up or remove it.
- A11Y [severity: medium]: amanda-repository/src/app/pages/project-detail/project-detail.component.html:25-31 — `<a>` nested inside `<button>` creates duplicate/invalid interactive semantics and confuses tab order — fix: use one or the other, not both.
- A11Y [severity: medium]: amanda-repository/src/app/pages/project-detail/project-detail.component.html:3-6 — Nav uses a hardcoded `class="active"` on "Work" regardless of actual route — misleading for all users, especially with assistive tech relying on `aria-current` — fix: bind active state to real route, or use `aria-current="page"`.
- QUALITY [severity: medium]: amanda-repository/src/app/pipes/safe-url.pipe.ts:9-11 — `SafeUrlPipe.transform()` unconditionally returns `null`; dead/broken code, imported but never applied — fix: implement it or delete it.
- QUALITY [severity: medium]: amanda-repository/src/app/pages/admin/admin.component.ts:17-172 — 173-line component mixing auth, data fetching, filtering, ~10 derived-metric getters, and formatting — exceeds 150-line max — fix: extract to a service/util module (see ANTI-PATTERN below).
- QUALITY [severity: medium]: amanda-repository/src/app/app.component.ts:6 — Unused `SafeUrlPipe` import — fix: remove.
- ANTI-PATTERN: amanda-repository/src/app/pages/admin/admin.component.ts:17-172 — God component (auth + fetching + filtering + metrics + formatting in one class) — fix: split into `AdminAuthService`, an analytics-aggregation module, and a thin presentation component.
- ANTI-PATTERN: amanda-repository/src/app/pages/admin/admin.component.html — Multiple `*ngFor` loops with no `trackBy`, iterating getters that return new object references every change-detection cycle — Angular tears down/recreates DOM nodes every cycle, breaking the `.bar-fill` width transition — fix: add `trackBy` and memoize the getters.
- ANTI-PATTERN: amanda-repository/src/app/pages/project-detail/project-detail.component.ts:27,29 — `projectData: any` / `Record<string, any>` defeats the `strict` mode set in tsconfig.json — fix: define a `Project` interface.
- QUALITY [severity: medium]: Calculator2.0/index.html:1604-1666, 1677-1881 — `drawMilestoneMarkers` (63 lines) and `lineChartUpdate` (~205 lines) both exceed the 30-line function max and 3-level nesting depth — fix: extract per-marker rendering and split bar/legend/scenario-line rendering into named helpers.
- QUALITY [severity: medium]: Calculator2.0/index.html:1399-1505 — `computeInsights` (~107 lines) mixes financial calculation logic with insight-object/UI construction — fix: split each insight into its own pure calc function, then a thin mapper builds the UI objects.
- SECURITY [severity: medium]: Calculator2.0/docs/decisions/2026-06-14-single-file-no-build-architecture.md — **Investigated and ruled a false positive.** A subagent reported a prompt-injection payload disguised as a system-reminder in this file. Direct inspection (`grep`, raw read) confirmed the file contains no such content — the subagent mistook a legitimate harness-generated reminder (appended after its own tool call, identical in form to ones seen elsewhere in this session) for injected file content. No action needed; noted here for the record since it was treated as a live security signal during the audit and is worth a second look if it recurs.

## 4. Dead Routes

- **amanda-repository**: Routes defined — `''` (MainLayoutComponent), `'project/:id'`, `'admin'`. All template hrefs resolve to `/` or in-page hash anchors on the root route. Programmatic `router.navigate(['/project', id])` calls were spot-checked against `projectMap` keys — all six project ids match. **No 404 risks found.**
- **Calculator2.0**: Non-SPA static page, no internal page-to-page links exist. Two 404 risks found in hardcoded references: `preview.png` (missing asset, breaks social previews) and the `feature/angular-makeover` branch reference in the deploy workflow (see Critical section — breaks CI/deploy, not a page route, but functionally the same class of dead-link risk).

## 5. Top 5 Action Items

1. **Rotate and stop committing secrets in amanda-repository.** Remove the hardcoded admin password and move auth server-side; verify Supabase RLS policies are airtight for the anon key in `environment.ts`.
2. **Fix the Calculator2.0 deploy workflow** before the next scheduled run — it checks out a deleted branch (`feature/angular-makeover`) and will fail, breaking the v1/v2 split deploy.
3. **Fix the failing amanda-repository test suite** — `ng test` doesn't even compile due to stale boilerplate in `app.component.spec.ts`, so no tests are currently running at all.
4. **Resolve the dual-root-component issue in amanda-repository** (`AppComponent` vs `RootComponent`, same selector) — SSR and client-rendered output are structurally different apps, which is a real correctness bug, not just style.
5. **Schedule the Angular major-version upgrade** (19→21) and triage the `npm audit` results (1 critical, 25 high) in amanda-repository — plan as a dedicated migration, not a routine bump.
