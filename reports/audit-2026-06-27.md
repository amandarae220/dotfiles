### 🟡 Medium

**Accessibility**

`A11Y [severity: medium]: tamagotchi-game/src/app/pages/setup/setup.component.html:41-53` — Buttons with `role="radio"` inside `role="radiogroup"` do not implement roving tabindex or arrow-key navigation. WAI-ARIA Authoring Practices require that only the selected radio is in the tab sequence (tabindex=0); arrow keys navigate between radios. As written, Tab cycles through all buttons and arrow keys do nothing, breaking the expected interaction model (WCAG 2.1 SC 2.1.1 Keyboard). — Fix: apply `tabindex="-1"` to unselected items, `tabindex="0"` to selected, and handle `keydown` ArrowLeft/ArrowRight to move focus and update selection.

`A11Y [severity: medium]: tamagotchi-game/src/app/pages/setup/setup.component.html:58` — Duplicate grouping semantics: a `<fieldset>` with `<legend>` already announces a group; adding `role="radiogroup"` on the inner `<div>` creates a second overlapping landmark for assistive technology. — Fix: remove the redundant `role="radiogroup"` and `aria-label` from the inner `<div>`.

`A11Y [severity: medium]: tamagotchi-game/src/app/pages/game/game.component.html:24` — `<label>{{ stat.label }}</label>` is used to display stat names (Fullness, Energy, Joy, Clean) without a `for` attribute or associated form control. Screen readers may announce these as orphaned labels. — Fix: replace `<label>` with `<span>` since these are display labels, not form-control labels.

`A11Y [severity: medium]: tamagotchi-game/src/app/pages/setup/setup.component.scss:102-103` — Input border `#d8d8d8` on background `#f4f4f4` yields a contrast ratio of ≈1.27:1, failing WCAG 1.4.11 Non-text Contrast (minimum 3:1 for UI component boundaries). The input boundary is not visually distinguishable in reduced-contrast environments. — Fix: darken border to at least `#aaaaaa` (≈3.1:1) or use `#999` to provide more headroom.

**Code Quality**

`QUALITY [severity: medium]: tamagotchi-game/src/app/services/pet-state.service.ts` — 174 lines exceeds the 150-line component/service limit. The service conflates profile management, stat transitions, persistence, and mood derivation in a single class. — Fix: extract `save()`/`load()` into a `PetPersistenceService` or move mood thresholds into `pet.models.ts`.

`QUALITY [severity: medium]: tamagotchi-game/src/app/pages/game/game.component.html:1` `tamagotchi-game/src/app/pages/setup/setup.component.html:1` — `[ngStyle]` for theming applies inline styles directly to the host element. Inline styles can interfere with forced-colors mode and user stylesheets (WCAG 1.4.4), and carry a specificity penalty that complicates overrides. — Fix: expose theme values as CSS custom properties on the component host using `@HostBinding('style')` with `--bg` and `--dot` variables, and consume them in SCSS.
