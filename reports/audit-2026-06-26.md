# Monthly Audit — 2026-06-26 (neo-control, where-it-counts)

## Executive Summary

Both repos build cleanly. neo-control's lint check fails on one ESLint error (`setState` inside a `useEffect`). The biggest risk items are: a client-side admin password check that exposes the passphrase in the JS bundle, a god-component `GameCanvas.tsx` at 3,126 lines, and two major-version dependency cliffs in where-it-counts (Svelte 4→5, vite 5→8). A typo in the Hero section copy also needs fixing.

## 1. Skill Compliance

### 🚨 Critical

**A11Y [severity: critical]: neo-control/src/pages/AdminPage.tsx:4 — Admin password compared client-side from `VITE_ADMIN_PASS`**
Any `VITE_` env var is bundled into the client JS and visible to anyone with DevTools. The passphrase is readable without any network sniffing. — Fix: move auth to a server function or use Supabase RLS + a JWT-gated endpoint.

**SECURITY [severity: critical]: neo-control/src/pages/AdminPage.tsx:4 — Admin passphrase exposed in client bundle**
`import.meta.env.VITE_ADMIN_PASS` is readable in the production bundle. String equality check against it provides no real access control. — Fix: validate the passphrase server-side (Supabase Edge Function or similar); never expose it as a `VITE_` variable.

### 🔴 High

**QUALITY [severity: high]: neo-control/src/game/GameCanvas.tsx:1–3126 — God component at 3,126 lines**
Game state, physics, rendering, input handling, networking, and UI overlays all live in a single file with a single `useEffect` loop. Functions like `update()` (lines 951–1302, ~350 lines) and `render()` (lines 1741–end) far exceed the 30-line function limit. — Fix: extract physics/game logic into `lib/gameLoop.ts`, draw routines into `lib/draw.ts`, and input handling into `lib/input.ts`. Keep the component as a thin orchestrator.

**QUALITY [severity: high]: neo-control/src/pages/AdminPage.tsx:1–1383 — File exceeds 150-line component limit**
`AdminPage.tsx` is 1,383 lines and contains 13+ components: `FunnelChart`, `FunnelSummary`, `BarChart`, `LineChart`, `PlayerProgressionCard`, `CohortRetentionTable`, `PathComparisonCard`, `PathFlowChart`, `Stat`, `ChartCard`, and `AdminPage` itself. — Fix: move each chart component to its own file under `src/components/admin/`.

**QUALITY [severity: high]: where-it-counts/src/lib/components/viz/SurplusMapViz.svelte:69–165 — `boot()` function is ~97 lines**
Single function handles D3 init, data loading, color scale setup, SVG construction, county rendering, and legend rendering. — Fix: split into `initProjection()`, `buildColorScales()`, `renderCounties()`, `renderLegend()`.

**QUALITY [severity: high]: where-it-counts/src/lib/components/viz/SurplusMapViz.svelte:208–296 — `addSwingLabels()` is ~88 lines**
Single function builds SVG leader lines, circles, and multi-line text labels for all three states. — Fix: extract a `mkStateLabel(svg, prefix, info, scale, tx, ty)` helper and loop over states.

**A11Y [severity: high]: neo-control/src/game/GameCanvas.tsx — Canvas element has no accessible name or role**
The `<canvas>` rendered by this component (see `useRef` + `useEffect` setup) has no `aria-label`, `role`, or `title`. Screen readers get a meaningless element. — Fix: add `aria-label="Neo Control arcade game — keyboard controls: arrow keys to move, Z to shoot, ESC to pause"` to the canvas element; add a visually-hidden live region for score updates.

**A11Y [severity: high]: neo-control/src/pages/AdminPage.tsx:111 — `FunnelChart` SVG missing accessible name**
The SVG element has no `role="img"` and no `aria-label`, making the entire funnel chart invisible to assistive technology. The path-flow SVG at line 916 does have `role="img" aria-label="..."` — apply the same pattern here. — Fix: add `role="img" aria-label="Wave progression funnel chart"` to the FunnelChart SVG.

**A11Y [severity: high]: where-it-counts/src/lib/components/viz/SurplusMapViz.svelte:301 — `console.error` in production boot**
`boot().catch(console.error)` leaks stack traces to the browser console in production and provides no user-facing fallback when the TopoJSON or surplus data fails to load. — Fix: catch the error, log to a monitoring service, and render an accessible error message in the container.

### 🟡 Medium

**A11Y [severity: medium]: where-it-counts/src/lib/components/sections/Calculator.svelte:289 — `aria-valuetext` is a static string that won't update**
```svelte
aria-valuetext="${budget.toLocaleString()} per month"
```
In Svelte, double-quoted attribute values with `${}` are plain strings — the interpolation never evaluates. Screen readers will always announce the literal text `${budget.toLocaleString()} per month`. — Fix: use `aria-valuetext="{budget.toLocaleString()} per month"` (single-quoted outer, curly braces for reactive binding).

**A11Y [severity: medium]: neo-control/src/game/SettingsPanel.tsx:42 — `cheatStatusRef` paragraph has no live region**
The cheat code status message is updated imperatively via `ref`, but has no `role="alert"` or `aria-live="polite"`, so screen-reader users never hear the result. — Fix: add `role="alert"` to the `<p ref={cheatStatusRef}>` element.

**A11Y [severity: medium]: neo-control/src/pages/LeaderboardOverlay.tsx:62 — Submit button missing `type="button"`**
The submit button inside a dialog has no explicit `type` attribute (defaults to `type="submit"` in some contexts). — Fix: add `type="button"` to the `ref={submitRef}` button.

**A11Y [severity: medium]: where-it-counts/src/lib/components/sections/Hero.svelte:5–8 — Copy typo in hero deck**
```
"…where the outcome where the margin was not in question."
```
"where" appears twice — likely meant "where the outcome **was** not in question" or "where the margin was not in question." This is visible to all users. — Fix: correct to `"…cast in places where the margin was not in question."`.

**QUALITY [severity: medium]: where-it-counts/src/lib/components/sections/Calculator.svelte — All destination data embedded inline**
173 lines of destination metro objects with rent, home price, margins, blurbs, and notes are hardcoded directly inside the component. — Fix: move to `src/lib/data/destinations.js` and import. Keeps the component under 150 lines and makes data changes auditable.

**QUALITY [severity: medium]: where-it-counts/src/lib/components/viz/SurplusMapViz.svelte:413 lines — Component over 150-line limit**
At 413 lines, the viz component contains D3 setup, data loading, color helpers, three separate render functions, and Svelte lifecycle hooks. — Fix: see `boot()` and `addSwingLabels()` extraction notes above.

**QUALITY [severity: medium]: where-it-counts/src/routes/+page.svelte:79 — Magic number color via inline style**
```svelte
<span class="stat" style="color:#6b7280;">582</span>
```
`#6b7280` is a hardcoded hex value that duplicates `--color-text-muted`. Violates the global token rule. — Fix: add a `.text-muted` utility class (already used elsewhere) and remove the inline style.

**ANTI-PATTERN: neo-control/src/pages/AdminPage.tsx:130,247,329 — Index keys in dynamic lists**
`<g key={i}>` used in `FunnelChart` (line 130), `BarChart` (line 247), and `LineChart` (line 329). Index keys cause React to reuse DOM nodes incorrectly when list items reorder or change. — Fix: use a stable key derived from the data (e.g., `key={stage.wave}`, `key={d.label}`).

**ANTI-PATTERN: neo-control/src/pages/AdminPage.tsx:1071–1073 — `setState` called synchronously inside `useEffect`**
```tsx
useEffect(() => {
  if (authed) load();   // load() calls setSessions, setError, setLoading
}, [authed]);
```
ESLint flags this as `react-hooks/set-state-in-effect`. The lint check fails because of this. — Fix: call `load()` directly after `setAuthed(true)` in the `attempt()` handler, and remove the effect.

**ANTI-PATTERN: neo-control/src/pages/AdminPage.tsx:1133–1138 — Derived state computed in render body**
`allDevices`, `allBrowsers`, `uniquePlayers`, `avgScore`, `avgWave`, and related aggregates are recomputed on every render from `filteredSessions`. — Fix: wrap in `useMemo` calls or compute once on data load.

`style={{ color: 'var(--green)' }}`, `style={{ textAlign: 'center' }}`, and similar props appear dozens of times across the file. — Fix: define semantic CSS classes in the admin stylesheet and replace inline styles.

**QUALITY [severity: low]: neo-control/src/game/GameCanvas.tsx — Magic numbers without named constants**
`W = 420`, `H = 560`, `PX = 3`, `PY = H - 52`, `P_SPD = 170`, `B_SPD = -300`, etc. are named at the top (good), but deeper in the file values like `0.28`, `0.55`, `44`, `2.2`, `1.8`, `15`, `140` appear without context. — Fix: name physics tuning constants (e.g., `GRAVITY_PULL_FACTOR = 0.28`, `DODGE_TOL_FRAMES = 44`).
